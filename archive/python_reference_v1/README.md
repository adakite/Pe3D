# Python Reference v1

This folder freezes the first end-to-end Python port before further parity work.

Contents:
- `src/g3point/`: snapshot of the working Python package and CLI
- `pyproject.toml`: package metadata used for the snapshot
- `outputs/Otira_1cm_grains.ply_granulo.csv`: baseline Otira granulometry CSV generated by the Python port
- `baseline_metrics.json`: matched comparison against the MATLAB Otira reference

Baseline command:

```bash
.venv/bin/g3point run --pointcloud G3point/PointCloud/Otira_1cm_grains.ply --output-root python_outputs_full
```

Baseline Otira summary:
- Segmented grains: `505` Python vs `503` MATLAB
- Granulometry rows: `400` Python vs `396` MATLAB
- Matched `Dmax` MAE: about `0.161 m`
- Matched `Dmed` MAE: about `0.110 m`
- Matched `Dmin` MAE: about `0.083 m`
- Orientation parity is still weak and is the main remaining gap
